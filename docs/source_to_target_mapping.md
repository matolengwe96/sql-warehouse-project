# Source-to-Target Mapping

**Version:** 1.0  
**Status:** Approved  
**Last Updated:** 2026-03-28

This document provides **column-level lineage** for every field as it moves through the Bronze, Silver, and Gold layers. It is the authoritative reference for writing, reviewing, and auditing ETL stored procedures.

---

## How to Read This Document

| Convention                   | Meaning                                                               |
| ---------------------------- | --------------------------------------------------------------------- |
| `—` in Transformation column | Value passes through unchanged from the previous layer                |
| `_(not in source)_`          | Column is generated by the ETL process, not present in the source     |
| `_(generated)_`              | Column is computed within the ETL (e.g. `ROW_NUMBER()`, `GETDATE()`)  |
| `_(derived from <col>)_`     | Column is derived by applying logic to another column in the same row |

**Layer summary:**

| Part | Mapping             | Load Strategy                               |
| ---- | ------------------- | ------------------------------------------- |
| 1    | Source CSV → Bronze | Full-truncate + `BULK INSERT`; no transform |
| 2    | Bronze → Silver     | Full-truncate + `INSERT`; clean, cast, std  |
| 3    | Silver → Gold       | View or full-refresh table; join + enrich   |

## Part 2: Bronze → Silver Layer

Silver procedures read exclusively from Bronze, apply cleansing rules, and write to Silver tables using `TRUNCATE` + `INSERT`. The Silver layer **never reads from source CSV files directly**.

**Universal Silver quality rules applied to all tables:**

| Rule                        | Implementation                                              |
| --------------------------- | ----------------------------------------------------------- |
| Trim whitespace             | `TRIM(column)` on all `NVARCHAR` columns                    |
| Replace blank with NULL     | `NULLIF(TRIM(column), '')`                                  |
| Standardise gender codes    | `CASE WHEN ... 'M' THEN 'Male' ... 'F' THEN 'Female' ...`   |
| Standardise marital status  | `CASE WHEN ... 'M' THEN 'Married' ... 'S' THEN 'Single'`    |
| Cast string dates           | `TRY_CAST(column AS DATE)` — NULL if unparseable            |
| Reject future birth dates   | `CASE WHEN bdate > GETDATE() THEN NULL`                     |
| Deduplicate on business key | `ROW_NUMBER() OVER (PARTITION BY <key> ORDER BY date DESC)` |
| Audit timestamps            | `dwh_created_at = GETDATE()`, `dwh_updated_at = GETDATE()`  |

### `bronze.erp_cust_az12` → `silver.erp_cust_az12`

| Bronze Column | Silver Column    | Transformation                                                                    | Rule                                                          |
| ------------- | ---------------- | --------------------------------------------------------------------------------- | ------------------------------------------------------------- |
| `cid`         | `cid`            | `REPLACE(cid, 'NAS-', '')`                                                        | Strip `NAS-` prefix if present                                |
| `bdate`       | `bdate`          | `TRY_CAST(bdate AS DATE)`                                                         | Cast string to DATE; NULL if unparseable; reject future dates |
| `gen`         | `gen`            | `CASE WHEN gen = 'M' THEN 'Male' WHEN gen = 'F' THEN 'Female' ELSE 'Unknown' END` | Standardise gender values                                     |
| _(generated)_ | `dwh_created_at` | `GETDATE()`                                                                       | Load timestamp                                                |
| _(generated)_ | `dwh_updated_at` | `GETDATE()`                                                                       | Load timestamp                                                |

**Quality rules:**

- Rows where `cid` is NULL or blank after prefix removal are excluded.
- Birth dates in the future (`bdate > CAST(GETDATE() AS DATE)`) are set to NULL.
